\section{LangGraph Orchestrator Flow}
The system is orchestrated using a centralised, versioned state buffer, managed via a cyclic directed graph in \textbf{LangGraph}. To prevent write conflicts and maintain transaction integrity across asynchronous agent steps, the graph state updates conform to a strict single-writer policy \cite{intellicode2025}. Specialist agents (such as the Socratic Tutor $A_t$ or simulated Student $A_s$) act as pure functions over read-only snapshots of the state buffer; only the Epistemic Tracker $A_e$ is authorised to commit write modifications back to the shared state buffer \cite{elhaimeur2026itas}. Furthermore, to investigate peer-agent social scaffolding dynamics (RQ2), the orchestrator stochastically instantiates a simulated peer student agent $A_{\text{peer}}$ alongside the target student $A_s$. This peer agent stochastically introduces controlled conceptual or arithmetic errors into the dialogue, triggering productive cognitive conflict and observational learning, which SOTA social learning research shows to increase learning outcomes \cite{kumar2026beyond}. As illustrated in Figure~\ref{fig:flowchart}, the active multi-agent cycle starts by running the Epistemic Tracker ($A_e$) and Socratic Tutor ($A_t$), before routing the output through a Pedagogical Guardrail ($A_g$). The guardrail checks if the tutor's response contains direct answers or raw target code (``Answer/Code Leaked?''); if it does, it intercepts the output and triggers a rewrite node to generate a Socratic question instead. Once a compliant Socratic response is produced, it is sent to the Simulated Student ($A_s$) to generate a response. The loop terminates when the maximum session round count is reached (``Max Turns Reached?'') or the student completes the target concepts.
